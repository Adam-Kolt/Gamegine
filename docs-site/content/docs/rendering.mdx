The renderer module provides a lightweight pygame window for visualising games, meshes,
paths, and trajectories during development. Any object implementing `Drawable` can be drawn
by the renderer.

## Renderer loop

`gamegine.render.renderer.Renderer` is a singleton responsible for window management and the
main draw loop. Typical usage:

```python
renderer = Renderer()
renderer.set_game(game)
renderer.set_render_scale(Centimeter(1.5))
renderer.init_display()

while renderer.loop():
    renderer.draw_static_elements()
    renderer.draw_element(path)
    renderer.draw_elements(extra_drawables)
    renderer.render_frame()

The renderer module now includes docstrings detailing each call—consult them if you need to
manually manage display state or tweak the animation tick.
```

## Colours and drawing helpers

`gamegine.render.helpers` contains utility functions for drawing rectangles, circles,
polygons, and generic discrete boundaries with a consistent visual style. Colours are
wrapped by `gamegine.render.style.Palette` together with `Shade` and `Opacity` enums for
quick variations.

### Composing scenes

- Use `renderer.draw_element` for single drawables and `renderer.draw_elements` for batches.
- Every helper in `render.helpers` accepts NCIM-aware measurements—a radius specified as
  `Inch(6)` will be converted automatically using the active render scale.
- The updated docstrings explain how each helper styles its shapes, making it easy to match
  the look of custom drawables to the built-in ones.

```python title="custom_drawable.py"
from gamegine.render.drawable import Drawable
from gamegine.render.helpers import draw_line, Palette

class VelocityVector(Drawable):
    def __init__(self, state):
        self.state = state

    def draw(self, render_scale):
        draw_line(
            self.state.x.get(),
            self.state.y.get(),
            self.state.x.get() + self.state.vel_x.get(),
            self.state.y.get() + self.state.vel_y.get(),
            render_scale * 0.1,
            Palette.GREEN,
            render_scale,
        )
```
